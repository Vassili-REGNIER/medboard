Classe utilitaire de validation et sanitization des entrées utilisateur

Fournit un ensemble complet de méthodes pour valider et nettoyer les données entrantes (formulaires, API, etc.). Chaque type de donnée (nom, email, username, password, etc.) dispose de méthodes de sanitization et de validation dédiées.

Les expressions régulières sont centralisées en constantes pour assurer la cohérence avec les contraintes SQL CHECK définies en base de données.

Pattern d'utilisation :

  1. Sanitize : nettoie et normalise la donnée (trim, lowercase, collapse spaces)
  2. Validate : vérifie la conformité et retourne un message d'erreur ou null

validateBase64UrlToken()

Valide un token au format base64url

public static validateBase64UrlToken(string $s[, int $min = 24 ][, int $max = 128 ][, string $label = 'Le token' ]) : string|null

Vérifie qu'un token :

  • N'est pas vide
  • Ne contient que des caractères base64url (A-Z, a-z, 0-9, -, _)
  • Respecte les contraintes de longueur min/max

Par défaut, un token de 32 octets fait ~43 caractères en base64url sans padding.
