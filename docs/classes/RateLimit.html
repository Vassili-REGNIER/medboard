Classe de limitation de débit (Rate Limiting)

Protège l'application contre les abus et attaques par force brute en limitant le nombre de tentatives autorisées pour une action donnée depuis une adresse IP.

Le système fonctionne avec une fenêtre de temps glissante : chaque action est comptabilisée et réinitialisée après expiration de la fenêtre.

Fonctionnalités principales :

  • Limitation par IP et par action (login, register, reset-password, etc.)
  • Fenêtre de temps configurable (par défaut 15 minutes)
  • Nombre maximal de tentatives configurable (par défaut 5)
  • Réinitialisation manuelle après succès
  • Compatible avec les proxies (support de X-Forwarded-For)

Le stockage se fait en session pour des raisons de simplicité. Pour une application à fort trafic, il serait préférable d'utiliser Redis ou Memcached.

check()

RateLimit.php : 60

Vérifie si une action est autorisée pour l'IP courante

public static check(string $action[, int $maxAttempts = 5 ][, int $windowSeconds = 900 ]) : bool

Implémente un système de fenêtre glissante :

  1. Récupère ou initialise le compteur pour l'action + IP
  2. Vérifie si la fenêtre de temps est expirée → réinitialise si oui
  3. Incrémente le compteur de tentatives
  4. Bloque si le nombre maximal de tentatives est dépassé

Structure de stockage en session :

$_SESSION["ratelimit_{action}_{ip}"] = [
    'count' => 3,                    // Nombre de tentatives
    'reset_at' => 1735689600         // Timestamp de réinitialisation
]

Exemple d'usage :

// Dans un contrôleur de login
if (!RateLimit::check('login', 5, 900)) {
    $remaining = RateLimit::getRemainingTime('login');
    Flash::set('errors', ["Trop de tentatives. Réessayez dans {$remaining}s"]);
    Http::redirect('/auth/login');
}
Parameters
$action : string

Identifiant de l'action à limiter (login, register, etc.)

$maxAttempts : int = 5

Nombre maximum de tentatives autorisées (défaut: 5)

$windowSeconds : int = 900

Durée de la fenêtre en secondes (défaut: 900 = 15 min)

Return values
bool —

True si l'action est autorisée, false si bloquée

getRemainingTime()

RateLimit.php : 160

Calcule le temps restant avant réinitialisation du compteur

public static getRemainingTime(string $action) : int

Retourne le nombre de secondes restantes avant que la fenêtre de limitation n'expire et que le compteur soit réinitialisé.

Utile pour afficher un message informatif à l'utilisateur bloqué, par exemple : "Trop de tentatives. Réessayez dans 12 minutes."

Exemple d'usage :

$remaining = RateLimit::getRemainingTime('login');
if ($remaining > 0) {
    $minutes = ceil($remaining / 60);
    Flash::set('errors', ["Réessayez dans {$minutes} minute(s)"]);
}
Parameters
$action : string

Identifiant de l'action

Return values
int —

Nombre de secondes restantes (0 si aucune limitation active)

reset()

Réinitialise le compteur de tentatives pour une action

public static reset(string $action) : void

Cette méthode doit être appelée après une action réussie (ex: après une connexion réussie) pour permettre à l'utilisateur de recommencer sans attendre l'expiration de la fenêtre.

Exemple d'usage :

// Après une connexion réussie
if ($loginSuccess) {
    RateLimit::reset('login');
    // ... suite du traitement
}
Parameters
$action : string

Identifiant de l'action à réinitialiser

getClientIp()

RateLimit.php : 127

Récupère l'adresse IP réelle du client

private static getClientIp() : string

Gère les cas où l'application est derrière un proxy ou un load balancer (comme sur AlwaysData). Vérifie d'abord le header X-Forwarded-For, puis utilise REMOTE_ADDR en fallback.

Pour X-Forwarded-For, prend uniquement la première IP de la liste (IP originale du client) pour éviter l'usurpation via injection d'IPs.

Return values
string —

L'adresse IP du client
